# 基建技能效率评分

此目录用于维护“基建技能一览”页面的效率排序分数。

页面会自动读取本目录下所有 `*.json` 文件，并对同一个技能 key 的所有有效分数取平均值。没有在任何评分文件中出现的技能，会在页面中按 `100` 分处理，使新增或遗漏的技能优先排到前面。

## 文件约定

- `auto.json`：由文本规则自动生成的基础分数。
- 其他评分文件：可由不同维护者自行新增，例如 `alice.json`、`bob.json`。
- 每个文件都是一个对象，key 为技能匹配键，value 为 `0` 到 `100` 的分数。

技能匹配键格式：

```text
name|roomType|buffName|phase|level
```

示例：

```json
{
  "裂响|manufacture|“连轴转”|2|1": 35,
  "凯尔希·思衡托|workshop|理论革新|2|1": 80
}
```

## auto.json 生成规则

`auto.json` 是根据 `src/static/json/build/building_table.json` 的技能描述文本自动估算出来的，规则只用于生成初始参考值，不代表最终人工判断。

生成时会先去掉描述中的 HTML 标签，然后提取正向数值：

- 百分比加成：例如 `生产力+35%` 记为 `35`。
- 带上限的“每个/每级”类效果：例如 `每级+10%，最多30%` 会取上限或估算上限值。
- 额外加成：例如 `+30%，额外+10%` 会估算为 `40`。
- 心情恢复：例如 `心情每小时恢复+0.55` 会换算为 `55`。
- 无法提取正向效率数值的技能记为 `0`。
- 最终分数会限制在 `0` 到 `100` 之间。

如果需要重新生成 `auto.json`，可用同样思路遍历 `building_table.json`：

1. 为每条技能生成 key：`name|roomType|buffName|phase|level`。
2. 清理 `description` 中的 HTML 标签。
3. 按上面的规则提取分数。
4. 输出为 key 到分数的 JSON 对象。

## 人工评分

人工评分不需要修改 `auto.json`。新增一个独立 JSON 文件即可，例如：

```json
{
  "裂响|manufacture|“连轴转”|2|1": 45
}
```

如果 `auto.json` 给了 `35`，另一个人工文件给了 `45`，页面实际使用分数为 `(35 + 45) / 2 = 40`。
