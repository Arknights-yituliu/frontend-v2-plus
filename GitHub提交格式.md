# Commit 格式

```text
提交类型(提交涉及的模块): 提交内容
```

例如

```text
fix(攒抽计算器): 每月黄票兑换月份错误
```

以下类型仅参考，可以用别的类型

| Commit 类型 |                          标题描述                          |
| :---------: | :--------------------------------------------------------: |
|    feat     |                           新功能                           |
|     fix     |                          修复 bug                          |
|    docs     |                          文档改变                          |
|   update    | 更新内容（小幅度改动，比如修改文本文本，更新 json 数据等） |
|     css     |                          更新样式                          |
|    style    |                        代码格式改变                        |
|  optimized  |                          优化代码                          |
|  refactor   |                      某个已有功能重构                      |
|    perf     |                          性能优化                          |
|    test     |                     增加测试/测试用例                      |
|   revert    |                    撤销上一次的 Commit                     |
|    chore    |                 杂项，不知道写啥可以写这个                 |
|    init     |                           新建库                           |
|    build    |            改变了构建工具，如 grunt 换成了 npm             |
|  resources  |                     静态资源（图片等）                     |
